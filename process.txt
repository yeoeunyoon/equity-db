===============================================================
process.txt — Data Acquisition and Cleaning Documentation
Equity Market Intelligence & Portfolio Analysis Database
===============================================================

OVERVIEW
--------
Data for this database is generated and cleaned using a Python script
(generate_data.py) that produces TSV files for bulk loading into MySQL.
Curated reference tables (sectors, industries, companies, exchanges,
securities, portfolios, holdings) are hard-coded from public references,
while price, financial snapshot, ETF, option, and corporate-action data
is pulled LIVE from Yahoo Finance via the yfinance Python library. The
25 selected S&P 500 companies, 5 major ETFs, and 5 sample option
contracts therefore reflect real market data over the requested date range.

DATA SOURCES
------------
1. Sector & Industry classifications
   - Source: S&P Global Industry Classification Standard (GICS)
     URL: https://www.spglobal.com/spdji/en/landing/topic/gics/
   - Format: Manually curated from public GICS reference tables
   - Cleaning: Selected 11 top-level sectors and 20 representative
     sub-industries. Mapped each of the 25 companies to its GICS
     industry. No duplicates or anomalies.

2. Company data
   - Source: Yahoo Finance company profiles
     URL: https://finance.yahoo.com/
   - Format: Company names, tickers, and exchange listings obtained
     from public profile pages.
   - Cleaning: Standardized company names (e.g., "Alphabet Inc."
     not "ALPHABET INC"). Verified each ticker maps to exactly one
     company. Country set to "US" for all (scope limited to US equities).

3. Exchange data
   - Source: General public knowledge (NYSE, NASDAQ, CBOE)
   - Format: Manually entered exchange names, countries, and IANA
     timezone identifiers.
   - Cleaning: None needed; only 3 exchanges.

4. Price (OHLCV) data
   - Source: Pulled LIVE from Yahoo Finance via the yfinance Python library.
     URL: https://pypi.org/project/yfinance/
     Command: yfinance.download(tickers, start=PRICE_START, end=PRICE_END,
                                group_by="ticker", auto_adjust=False)
   - Format: yfinance returns a pandas DataFrame with columns:
     Date, Open, High, Low, Close, Adj Close, Volume
   - Date range: driven by the --start/--end CLI flags (or PRICE_START/
     PRICE_END env vars); defaults to the last 3 years through today.
   - Cleaning steps applied to the live yfinance data:
     * Keep only Open, High, Low, Close, Volume (Adj Close not in schema)
     * Use the DataFrame index (Date) as trade_date
     * Drop rows with NaN Close (holidays where the market was closed)
     * Round prices to 4 decimal places
     * Coerce volume to a non-negative integer (0 when missing)
     * Map ticker to security_id via lookup table

5. Financial Snapshot data
   - Source: Pulled LIVE from the yfinance Ticker.info dict.
     Fields: marketCap, trailingPE, trailingEps, beta
   - Cleaning: pe_ratio rounded to 2 decimals, eps/beta to 4 decimals;
     missing fields stored as NULL. One snapshot per security, dated with
     the --end date (SNAPSHOT_DATE).

6. Corporate Action data
   - Source: Pulled LIVE from yfinance Ticker.dividends and Ticker.splits
     history, filtered to the requested PRICE_START..PRICE_END window.
   - Cleaning: Dividends recorded as action_type 'dividend'; splits mapped
     to 'split' (ratio > 1) or 'reverse_split' (ratio < 1) to satisfy the
     ENUM constraint. Amounts rounded to 6 decimals.

7. Portfolio & Holding data
   - Source: Fabricated for demonstration purposes.
   - 5 portfolios with distinct investment strategies (growth, value,
     tech, dividend income, balanced).
   - 5 holdings per portfolio (25 total), with realistic share counts
     and cost basis values.
   - Cleaning: Verified all security_id references exist in Security
     table. Verified shares > 0 and average_cost >= 0.

8. ETF data
   - Source: Fund name and expense ratio pulled LIVE from the yfinance
     Ticker.info dict (longName/shortName, annualReportExpenseRatio),
     with hard-coded fallbacks used only if the live lookup fails.
   - Cleaning: Expense ratios expressed as percentages (e.g., 0.0945
     for SPY = 9.45 basis points). Verified non-negative.

9. Option Contract data
   - Source: 5 sample contracts (calls/puts on AAPL, MSFT, TSLA, SPY),
     ticker symbology per OCC standard format. Implied volatility and
     open interest are pulled LIVE from the yfinance option_chain() for
     the matching strike/expiration, with hard-coded fallbacks used only
     if the live lookup fails.
   - Cleaning: Verified strike_price > 0, open_interest >= 0.

FILE FORMAT
-----------
All output files are tab-separated (TSV) with a header row.
Encoding: UTF-8
Line endings: Unix (LF)
Date format: ISO 8601 (YYYY-MM-DD)
Decimal separator: period (.)
NULL values: represented as empty string

SCRIPT USAGE
------------
0. Install dependencies: pip install -r requirements.txt   (yfinance, pandas)

1. Run (defaults to the last 3 years of live data through today):
     python3 generate_data.py

   Choose an explicit price date range via CLI flags:
     python3 generate_data.py --start 2024-01-01 --end 2024-12-31

   ...or via environment variables (CLI flags take precedence):
     PRICE_START=2024-01-01 PRICE_END=2024-12-31 python3 generate_data.py

   Notes:
     * --end (env PRICE_END) defaults to today.
     * --start (env PRICE_START) defaults to three years before today.
     * SNAPSHOT_DATE is always set to the end date.

2. Output: 13 TSV files in ./data/ directory.
3. Load:  mysql -u <user> -p <db> --local-infile=1 < load.sql
